Changelog

The most up-to-date-sources are available via bci2000.org's svn repository. Here we list the (possibly less frequent) demo builds that we make available on our download page. See also our page in the BCI2000 wiki.

2011-07-10: v. 33960

Fifth release of the demo build. Since I've now moved to bci2000.org as the one-and-only place to version-control the Python sources, BCPy2000 revision numbers are henceforth equal to the corresponding bci2000.org svn revision numbers times 10 (to keep them strictly increasing since previous versions).

  • Various bugfixes and minor feature extensions
  • Downloads now contain BCI2000 v3.0 binaries

2010-07-08: v. 17374

Fourth release of the demo build, corresponding to bci2000.org revision number 2898.

  • Various bugfixes and tuneups
  • New API methods changed, detect_event, require_version and lock_transitions.
  • Various new tools, including TrapSequence and TriggerlessTrapSequence which are useful for ERP-based BCIs.

2009-09-22: v. 13637

Third release of the demo build, corresponding to bci2000.org revision number 2551.

  • We now have an additional strategy for packaging BCPy2000 for delivery: we include a full, portable, runnable, Python distro preconfigured with all the necessary packages, including the BCPy2000 framework. In this release, it contains NumPy 1.3.0, VisionEgg 1.2.1 and SciPy 0.7.1 among other recent advances. See the download page for details.
  • All VisionEgg- and pygame-specific code is now factored out into VisionEggRenderer.py. This paves the way for developers to write their own replacements, to remove the dependency on VisionEgg and OpenGL if desired. Whatever renderer subclass is implemented, an instance of it is available as self.screenâ€”see the renderer documentation.
  • In keeping with the feel of self.stimulus() and self.phase() calls, parameter and state definitions do not now have to be return arguments of self.Construct(): the new self.define_param() and self.define_state() API calls may be used instead.
  • Various bugfixes and improvements.

2009-03-16: v. 11336

Second release of the demo build. Version numbering of the BCPy2000 Python package now uses the revision numbers from the MPI subversion repository, where we manage our development copy. This means we have jumped from version 1.0 to version 11336, which is clearly quite some progress. The bci2000.org revision number corresponding to this release is 2324.

  • BCPy2000 modules are now all gnu-compilable and can be made to run on non-Windows systems.
  • In the application module, the VisionEgg main loop has been rearranged and streamlined, in the continuing quest for more reliable video timing.
  • Robuster parameter interpretation in BCI2000Tools/FileReader.py, and a fix in BCI2000Tools/PlaybackSourceModule.py fix errors that occurred when playing back some types of data file (reported by Marco Rotonda).
  • In the application module when running under Windows, two bits of automatic housekeeping are performed. On StartRun the new self.focus() method is used to raise the VisionEgg window, and the new self.volume() method is used to set the system volume to maximum. In StopRun. the same methods are called to re-raise the Operator window and set the system volume back to what it was at initialization time.
  • A bug-fix in WavTools makes the timing of sound stimuli more precise when played using the PyAudio interface. Also, panning is more flexible, and the default method of normalizing different channels' gains has changed, meaning that centre-panned sounds are louder by default than they used to be.
  • In SigTools, more flexible wavegen replaces sinewave.
  • Numerous small bug-fixes and enhancements.

2008-09-30: v. 8570

Known at the time as 1.0, this was our first release. The framework was intact, documented and usable for programming all three kinds of BCI2000 core module in Python. The additional tools were usable, though incompletely documented. The bci2000.org revision number corresponding to this release is 2150. A page was also created for this project at mloss.org.
